# Functional Requirements

This document lists the core functional requirements (FR) for the Connect Digital School platform.

| ID | Feature | Description |
|---|---|---|
| **FR-001** | **User Authentication** | Email/Password and OAuth (Google/Facebook) via Better Auth. Email verification and password reset required. |
| **FR-002** | **Course Management** | CRUD for courses, modules, and lessons. Supports video uploads and quiz attachments. |
| **FR-003** | **Video System** | Adaptive bitrate streaming (HLS/DASH). Progress tracking synced every 30s. |
| **FR-004** | **Assessment System** | Multiple question types (MCQ, Fill-blank, Listening, Speaking). Cooldown for retakes. |
| **FR-005** | **Payments** | Mercado Pago integration for PIX, Credit, and Debit. Webhook-driven status updates. |
| **FR-006** | **Email Notifications** | Automated emails for welcome, payment, certificates, and reminders via Resend. |
| **FR-007** | **Gamification** | Points, 30+ badges, student levels, daily streaks, and leaderboards. |
| **FR-008** | **Forum/Community** | Discussion threads, nested replies, upvoting, and teacher moderation tools. |
| **FR-009** | **Progress Tracking** | Visual progress bars, time-spent analysis, and personalized recommendations. |
| **FR-010** | **Admin Dashboard** | Content analytics, revenue reports, user management, and platform settings. |
| **FR-011** | **Certificate System** | PDF generation with unique verification codes and public verification page. |
| **FR-012** | **Landing Page** | SEO-optimized public site with hero section, pricing, and FAQ. |
